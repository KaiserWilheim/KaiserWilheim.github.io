$Base64$

$Base64$ 算法是一种广泛运用的加密算法。
其原理是这样的：

  1. 选取三位字符；
  2. 转为ASCII码；
  3. 拆分为4个6位长度的二进制数，并在每个数前加入两个前导零；
  4. 按照表格转换为对应的字符；
  5. 如果不够四个字符长就在后面补=。

因为每个转换后的二进制数有两个前导零，所以我们只需要64个对应的字符就可以完成加密。

范围 $[0,25]$ 、 $[26,51]$ 、 $[52,61]$ 分别对应 $A$-$Z$ 、 $a$-$z$ 、 $0$-$9$ 这些字符。
数字62与63分别对应+与/。

举个栗子：

$CSP$ 这三个字符分别对应的ASCII码是$67$ 、 $83$ 与 $80$ ，
转换为二进制后分别为 $01000011$ 、 $01010011$ 与 $01010000$ 。
按每六位拆分，可得结果为 $010000$、$110101$、$001101$与$010000$；
再次转换为10进制，得 $16$、$52$、$13$与$16$；
分别对应表中的 $Q0NQ$ 四个字符。

是不是很简单？

初始哈希值

初始哈希值 $H^{(0)}$ 取自前八个质数的平方根小数部分的前32位。
例如： $H^{(0)}_1$ 取 $\sqrt[]{2}$ 的小数部分前32位，即0x6a09e667。

预处理

在对信息进行加密之前，先要对信息进行预处理。

补码

我们首先在消息的末尾补上一个1.
接下来再补上 $k$ 个0，其中 $k$ 为满足下列方程的最小正整数：
\begin{equation}
l+1+k\equiv 448 \pmod{512}
\end{equation}
（这就意味着，当二进制串的长度 $\bmod{512} \equiv 448$ 时，我们要在最后补上512个0）
接着，我们将会在后面补上原消息的长度，以64位二进制数来表示。

现在我们以“CSP 2021 j/s”为例，作为栗子：

  • 首先转换为ASCII码，以二进制表示。

因此，原消息“CSP 2021 j/s”转换以后是 01000011 01010011 01010000 00100000 00110010 00110000 00110010 00110001 00100000 01101010 00101111 01110011
我们先补上一个1，之后再补上448-96-1=351个0，最后再加上原消息的长度——96位，结果用16进制表示是这样的：

拆分

之后，我们将补充完成后的消息以512位为单位拆分成块，其中第 $i$ 个消息块称为 $M^{(i)}$ 。
每一个消息块又以32位为单位进行拆分成为更小的块，其中第 $i$ 块的第 $j$ 小块称为 $M^{(i)}_{j-1}$ 。

此时，我们的预处理就完成了。

逻辑函数

$SHA256$ 算法用到了6个大量的逻辑函数，这里列举一下：

$Ch(x,y,z) = (x \land y) \oplus ( \neg x \land z) $
$M_{aj}(x,y,z) = (x \land y) \oplus (x \land z) \oplus (y \land z) $
$Σ_0(x) = S^2 (x) \oplus S^{13} (x) \oplus S^{22} (x) $
$Σ_1(x) = S^6 (x) \oplus S^{11} (x) \oplus S^{25} (x) $
$σ_0(x) = S^7 (x) \oplus S^{18} (x) \oplus R^{23} (x) $
$σ_1(x) = S^{17} (x) \oplus S^{19} (x) \oplus R^{10} (x) $

\begin{align}
Ch(x,y,z) & = (x \land y) \oplus ( \neg x \land z) \
M_{aj}(x,y,z) & = (x \land y) \oplus (x \land z) \oplus (y \land z) \
Σ_0(x) & = S^2 (x) \oplus S^{13} (x) \oplus S^{22} (x) \
Σ_1(x) & = S^6 (x) \oplus S^{11} (x) \oplus S^{25} (x) \
σ_0(x) & = S^7 (x) \oplus S^{18} (x) \oplus R^{23} (x) \
σ_1(x) & = S^{17} (x) \oplus S^{19} (x) \oplus R^{10} (x)
\end{align}
